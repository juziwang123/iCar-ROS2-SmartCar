# iCar 智能小车 ROS2 项目测试报告

## 1. 文档说明

本文档用于说明 iCar 智能小车 ROS2 项目的测试目标、测试范围、测试环境、测试用例设计、当前测试结果记录方式及后续补充计划。由于当前仓库仍处于功能骨架和文档完善阶段，本文档同时承担“测试计划 + 测试记录模板”的作用，便于后续联调时直接补充真实结果。

## 2. 测试目标

1. 验证系统各模块是否满足需求分析报告中的核心要求。
2. 验证底盘控制、雷达感知、视觉识别和导航模块能否独立运行。
3. 验证多模块联调时通信链路是否畅通，是否存在冲突控制。
4. 验证项目在实训演示场景下的可操作性、稳定性和安全性。

## 3. 测试范围

### 3.1 本次覆盖范围

1. 远程连接与环境准备。
2. ROS2 基础命令与节点/话题可见性验证。
3. 底盘控制与手动遥控。
4. 激光雷达避障、跟随或警卫功能。
5. 建图与导航流程。
6. 视觉检测与追踪。
7. 文档与交付物完整性。

### 3.2 暂不覆盖范围

1. 多车协同。
2. 云平台持续集成部署。
3. 长时间无人值守稳定性测试。

## 4. 测试环境

| 类别 | 说明 |
|------|------|
| 开发环境 | Windows 11 + WSL2 Ubuntu 22.04 |
| 本地 ROS 环境 | ROS2 Humble |
| 实车环境 | Jetson Ubuntu 20.04 + ROS2 Foxy |
| 硬件 | iCar 小车、激光雷达、深度相机、底盘控制板 |
| 网络 | 手机热点或局域网 |
| 工具 | SSH、NoMachine、VNC、VS Code |

## 5. 测试策略

### 5.1 测试分层

1. 环境验证：确认网络、登录、远程桌面和 ROS2 基础环境正常。
2. 单模块测试：分别验证控制、雷达、视觉、导航模块。
3. 集成测试：验证多模块同时运行时的话题交互和控制优先级。
4. 场景测试：围绕答辩演示路线设计完整流程。

### 5.2 测试方法

1. 命令行检查：使用 ros2 node list、ros2 topic list 等命令确认运行状态。
2. 功能观察：通过实车运动、RViz 画面、日志输出确认是否达到预期。
3. 异常注入：通过断网、遮挡、障碍物接近等方式验证容错能力。
4. 文档核对：验证使用手册能否指导他人复现实验流程。

## 8. 当前测试结论

1. 把每个功能模块的最小可运行链路尽快跑通。
2. 在实际联调时将测试结果、截图、日志和问题清单及时补入本报告。
