# iCar 智能小车 ROS2 项目使用手册

## 1. 手册说明

本文档面向项目组成员、指导教师和后续接手人员，说明 iCar 智能小车 ROS2 项目的环境准备、设备连接、常用命令、基础运行流程与故障排查方法。本文档以当前仓库规划和现有连接资料为基础编写，后续应随着功能实现持续补充具体启动命令与演示流程。

## 2. 使用前准备

### 2.1 硬件准备

1. iCar 智能小车主机。
2. 激光雷达、深度相机、底盘控制板及其连接线。
3. 供电电池或充电设备。
4. Windows 笔记本电脑。
5. 手机热点或可用局域网。

### 2.2 软件准备

1. Windows 11。
2. WSL2 Ubuntu 22.04。
3. ROS2 Humble 开发环境。
4. VS Code。
5. NoMachine 客户端或 VNC Viewer。
6. Git。

### 2.3 账号与连接信息

默认参考信息如下：

| 项目 | 内容 |
|------|------|
| 小车登录用户名 | jetson |
| 小车登录密码 | yahboom |
| 连接方式 | SSH / NoMachine / VNC |

如果现场设备已被重新配置，应以当日实际分配信息为准。

## 3. 开发环境准备

### 3.1 克隆项目仓库

在开发机中执行：

```bash
git clone git@github.com:juziwang123/iCar-ROS2-SmartCar.git
cd iCar-ROS2-SmartCar
```

### 3.2 本地 ROS2 工作环境

建议在 Ubuntu 22.04 + ROS2 Humble 中开发。示例流程如下：

```bash
mkdir -p ~/ros2_ws/src
cd ~/ros2_ws/src
git clone git@github.com:juziwang123/iCar-ROS2-SmartCar.git
cd ..
colcon build
source install/setup.bash
```

说明：如果仓库直接作为工作空间 src 目录使用，应根据实际目录调整 clone 位置。

### 3.3 注意事项

1. 不要将 ROS2 Foxy 与 Humble 混用在同一工作空间中。
2. 本地开发环境与实车运行环境存在差异，部署前需要再次确认依赖。
3. 实车调试前，建议先在本地验证命令、参数和流程。

## 4. 小车连接流程

### 4.1 连接同一网络

1. 打开手机热点或接入同一局域网。
2. 让小车与开发电脑连接到同一网络。

### 4.2 获取小车 IP

在小车终端执行：

```bash
ifconfig wlan0
```

或：

```bash
ip addr show wlan0
```

记下 inet 对应的 IP 地址。

### 4.3 网络连通性检查

在 Windows PowerShell 中执行：

```powershell
ping <小车实际IP>
```

若 ping 不通，优先检查是否连接到了同一热点。

### 4.4 SSH 登录

```powershell
ssh jetson@<小车实际IP>
```

输入密码后进入小车命令行环境。

### 4.5 NoMachine 远程桌面

1. 确认小车已安装 NoMachine 服务端。
2. 打开 Windows 端 NoMachine 客户端。
3. 新建连接并输入小车 IP。
4. 使用设备账号密码登录。

如果 NoMachine 不可用，可尝试 VNC 作为备选方案。

## 7. 基础运行流程

### 7.1 启动前检查

1. 确认小车电量充足。
2. 确认雷达、相机、底盘线缆连接可靠。
3. 确认开发电脑与小车网络互通。
4. 确认 ROS2 环境已 source。

### 7.2 推荐启动顺序

建议按以下顺序启动系统：

1. 启动小车并登录系统。
2. 启动底盘相关基础服务。
3. 启动激光雷达或相机驱动。
4. 启动建图、导航、避障或视觉模块。
5. 最后启动上位机控制或演示模块。

说明：当前仓库核心代码仍在逐步补充，具体 launch 命令应以实际实现后的 README 或功能包说明为准。

### 7.3 手动控制建议

1. 首次运行先验证停止命令是否有效。
2. 保持低速调试，避免在复杂环境中直接高速运行。
3. 有自动控制功能运行时，保留人工接管手段。

## 8. 小车基础功能操作

小车预置了两套 Docker 容器，覆盖建图、导航、避障、跟随、追踪等完整功能。

### 8.1 容器 A：icar/ros-foxy:1.0.2（基础控制容器）

启动：`./run_docker.sh`  
快捷指令：`is`（启动容器）、`it`（停止容器）、`id`（进入容器）

进入容器后可用指令：

| 指令 | 完整命令 | 用途 |
|------|---------|------|
| `n1` | `ros2 run icar_bringup Mcnamu_driver_X3` | 底盘驱动 |
| `n2` | `ros2 launch sllidar_ros2 sllidar_launch.py` | 雷达驱动 |
| `n3` | `ros2 run icar_laser laser_Avoidance_a1_X3` | 雷达避障 |
| `n4` | `ros2 run icar_laser laser_Tracker_a1_X3` | 雷达跟随 |
| `n5` | `ros2 run icar_laser laser_Warning_a1_X3` | 雷达警卫 |
| `m1` | `ros2 launch astra_camera astra.launch.xml` | 深度相机 |
| `m2` | `ros2 run icar_astra colorHSV` | HSV 颜色识别 |
| `m3` | `ros2 run icar_astra colorTracker` | 颜色追踪 |

### 8.2 容器 B：自动导航容器

启动：`./run_docker_autodrive.sh`  
快捷指令：`s`（启动容器）、`t`（停止容器）、`d`（进入容器）

**建图流程（m1-m4）：**

| 指令 | 完整命令 | 用途 |
|------|---------|------|
| `m1` | `ros2 launch yahboomcar_nav map_gmapping_launch.py` | 启动 gmapping 建图 |
| `m2` | `ros2 launch yahboomcar_nav display_map_launch.py` | RViz 地图展示 |
| `m3` | `ros2 run yahboomcar_ctrl yahboom_keyboard` | 键盘控制（键位：u/i/o/j/k/l/m/,/.） |
| `m4` | `ros2 launch yahboomcar_nav save_map_launch.py` | 保存地图（退出时按提示输入 y/n） |

**导航流程（n1-n4）：**

| 指令 | 完整命令 | 用途 |
|------|---------|------|
| `n1` | `ros2 launch yahboomcar_nav laser_bringup_launch.py` | 导航基础节点 |
| `n2` | `ros2 launch yahboomcar_nav display_nav_launch.py` | RViz 导航展示 |
| `n3` | `ros2 launch yahboomcar_nav navigation_dwa_launch.py` | DWA 路径规划 |
| `n4` | `ros2 launch yahboomcar_nav navigation_teb_launch.py` | TEB 路径规划 |

### 8.3 建图操作步骤

1. 启动自动导航容器：`./run_docker_autodrive.sh`
2. 开三个终端，每个执行 `d` 进入容器
3. 终端1：`m1`（建图基础）
4. 终端2：`m2`（RViz）
5. 终端3：`m3`（键盘控制，用 u/i/o/j/k/l/m/,/. 开车）
6. 绕场一周后，终端3 `Ctrl+C`，执行 `m4` 保存地图

### 8.4 避障/跟随/警卫操作步骤

1. 启动基础控制容器：`./run_docker.sh`
2. 终端1：`id && n1`（底盘驱动）
3. 终端2：`id && n2`（雷达驱动）
4. 终端3：`id && n3`（避障）或 `n4`（跟随）或 `n5`（警卫）

## 9. 故障排查

| 问题 | 可能原因 | 处理建议 |
|------|----------|----------|
| Ping 不通 | 不在同一网络 | 重新连接同一热点或局域网 |
| SSH 登录失败 | 账号密码错误或服务异常 | 核对用户名密码，检查网络和 SSH 服务 |
| 远程桌面卡顿 | 网络质量差 | 降低显示质量或切换到命令行调试 |
| ROS2 命令不可用 | 环境未 source | 重新执行 source install/setup.bash |
| 节点启动失败 | 依赖缺失或版本不匹配 | 检查日志、环境版本和依赖安装情况 |
| 小车不动 | 底盘服务未启动、控制链路未打通 | 先验证底盘基础驱动，再检查 /cmd_vel 流程 |
| 雷达/相机无数据 | 设备未识别或驱动未启动 | 检查 USB/串口连接与驱动状态 |

## 10. 安全注意事项

1. 所有运动调试必须在开阔、安全、可控区域进行。
2. 首次运行新功能时必须限速，并安排一人专门负责急停。
3. 禁止在人员密集区域进行自动导航和跟随实验。
4. 调试前确认设备供电稳定，避免低电量导致异常行为。
