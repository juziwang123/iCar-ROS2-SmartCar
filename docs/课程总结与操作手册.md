# iCar 智能小车 ROS2 项目 — 课程总结与操作手册

> 基于"智能小车课程材料"六大模块和现有项目文档，梳理我们要做什么、怎么做、每一步的具体命令。

---

## 一、项目总体目标

基于 **iCar 智能小车**（Jetson Orin Nano + ROS2 Foxy + 思岚 A1 激光雷达 + AstraPro Plus 深度相机 + AT32 底盘 + 8 个环境传感器），在实训周期内完成一个**可演示、可答辩**的智能小车系统，覆盖以下核心能力：

| 能力域 | 说明 |
|--------|------|
| 远程连接与运维 | SSH/NoMachine 连接小车，管理 ROS2 节点 |
| 底盘运动控制 | 键盘遥控、APP 遥控、速度/方向控制、急停 |
| 激光雷达 SLAM 建图 | 在实验室环境中建图并保存地图 |
| 自主导航 | 加载地图后单点/多点导航 |
| 激光雷达避障 | 检测障碍物自动减速/停车 |
| 激光雷达跟随/警卫 | 跟随目标、区域警戒 |
| 视觉目标检测 | 颜色识别、YOLO 目标检测 |
| 视觉追踪/循线 | 相机追踪目标或沿引导线行驶 |
| APP 远程控制 | 鸿蒙 APP 通过 HTTP/WebSocket 控制小车 |
| 高阶场景（加分项） | 道面缺陷检测、自然语言任务切换、多车协同等 |

## 三、操作手册

### 3.1 环境准备

#### 开发机环境（Windows + WSL2）

```bash
# 1. WSL2 中安装 ROS2 Humble（略，参考官方文档）
# 2. 克隆项目
cd ~
git clone git@github.com:juziwang123/iCar-ROS2-SmartCar.git
cd iCar-ROS2-SmartCar

# 3. 构建
colcon build --symlink-install
source install/setup.bash
```

#### 实车环境（Jetson Orin Nano, Ubuntu 20.04 + ROS2 Foxy）

小车已预置两套 Docker 容器：
- **icar/ros-foxy:1.0.2**：基础控制（雷达、避障、跟随、警卫、相机、颜色追踪）
- **导航容器**：gmapping 建图、DWA/TEB 导航

### 3.2 小车连接流程

```bash
# 1. 手机开热点，小车和电脑连同一热点

# 2. 在小车终端查看 IP
ifconfig wlan0

ip addr show wlan0

# 3. 从电脑 ping 小车

# 4. SSH 登录
ssh jetson@<小车IP>
# 密码: yahboom

# 5. 或者用 NoMachine 远程桌面
# 打开 NoMachine → 新建连接 → 输入小车IP → 账号 jetson / yahboom
```

### 3.5 我们项目的启动流程

#### 3.5.1 仿真环境（本地验证用）

```bash
# 终端1：启动仿真
source install/setup.bash
ros2 launch wpr_simulation2 wpb_simple.launch.py

# 终端2：启动 bringup
source install/setup.bash
ros2 launch car_bringup bringup.launch.py use_keyboard:=false use_lidar_avoidance:=true

# 终端3：发布手动控制指令
ros2 topic pub /cmd_vel_manual geometry_msgs/msg/Twist "{linear: {x: 0.2}, angular: {z: 0.0}}"
```

#### 3.5.2 建图流程

```bash
# 1. 启动仿真 + bringup
# 终端1
ros2 launch wpr_simulation2 wpb_simple.launch.py

# 终端2：启动建图模式
ros2 launch car_bringup bringup.launch.py mapping:=true

# 终端3：键盘遥控小车建图
ros2 run car_control keyboard_teleop

# 2. 保存地图
ros2 run nav2_map_server map_saver_cli -f ~/maps/lab_map

# 3. 停止建图
# Ctrl+C 关闭建图 launch
```

#### 3.5.3 导航流程

```bash
# 终端1：仿真
ros2 launch wpr_simulation2 wpb_simple.launch.py

# 终端2：导航模式（加载之前保存的地图）
ros2 launch car_bringup bringup.launch.py navigation:=true map_file:=/home/root/maps/lab_map.yaml

# 终端3：发布导航目标点
ros2 run car_navigation goal_publisher
# 或手动发布
ros2 topic pub /goal_pose geometry_msgs/msg/PoseStamped "{pose: {position: {x: 2.0, y: 1.0}}}"
```

#### 3.5.4 避障模式

```bash
# 终端1：仿真
ros2 launch wpr_simulation2 wpb_simple.launch.py

# 终端2：启动避障
ros2 launch car_bringup bringup.launch.py use_lidar_avoidance:=true use_keyboard:=true

# 终端3：键盘控制，小车前方有障碍物会自动减速/停车
```

### 3.6 视觉模块开发（car_vision）

#### 3.6.1 YOLO 目标检测部署

```bash
# 1. 进入小车 Docker
d

# 2. 安装 ultralytics
pip3 install ultralytics

# 3. 启动视觉节点（待开发完成后）
ros2 run car_vision yolo_detector --ros-args -p model:=yolov8n.pt
```

#### 3.6.2 设计要点

- 订阅 `/camera/color/image_raw`（RGB 图像）
- 使用 YOLOv8n 模型（6MB，CPU 可跑 30+ FPS）
- 发布 `/detections`（检测结果）+ `/cmd_vel_vision`（控制指令）
- 控制逻辑：前方有人 → 减速停车，有 stop sign → 停车 3 秒

### 3.7 APP 桥接开发（car_app_bridge）

#### 后端接口（已由小车 Flask 后端提供）

```bash
# 基础地址
http://<小车IP>:5000

# 获取状态
GET /api/state

# 发送控制指令
POST /api/cmd
{"type": "move", "linear": 0.2, "angular": 0.0}
{"type": "stop"}
{"type": "mode", "mode": "nav"}          # manual/nav/vision/follow
{"type": "estop", "active": true}

# 进程管理
POST /api/process/start
{"function": "mapping"}                  # mapping/navigation/avoidance/follow/guard
```

#### 鸿蒙 APP 四 Tab 设计

| Tab | 功能 |
|-----|------|
| Tab1 遥控 | 方向按钮、急停、模式选择、速度滑块 |
| Tab2 建图&导航 | 开始/停止建图、保存地图、开始/停止导航 |
| Tab3 雷达&视觉 | 避障/跟随/警卫开关、相机/颜色识别/追踪开关 |
| Tab4 状态 | 实时模式、速度、急停状态、运行节点列表 |

### 3.8 常用调试命令

```bash
# ROS2 基础
ros2 node list          # 查看所有节点
ros2 topic list         # 查看所有话题
ros2 topic echo /cmd_vel      # 监听速度指令
ros2 topic echo /scan         # 监听雷达数据
ros2 topic info /scan         # 查看话题类型

# 查看节点图
ros2 run rqt_graph rqt_graph

# 手动发布控制指令
ros2 topic pub /cmd_vel_manual geometry_msgs/msg/Twist \
  "{linear: {x: 0.2, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"

# 设置导航目标点
ros2 topic pub /goal_pose geometry_msgs/msg/PoseStamped \
  "{header: {frame_id: 'map'}, pose: {position: {x: 2.0, y: 0.0, z: 0.0}, orientation: {w: 1.0}}}"

# 保存地图
ros2 run nav2_map_server map_saver_cli -f ~/my_map

# 雷达串口赋权限（如遇权限问题）
sudo chmod 777 /dev/ttyUSB0
```

## 五、注意事项

1. **开发环境与实车环境不同**：本地 ROS2 Humble（Ubuntu 22.04），实车 ROS2 Foxy（Ubuntu 20.04）。使用标准消息接口，避免版本兼容问题。
2. **固件必须匹配**：ROS 功能用 `iCar.bin`，循线演示用 `follow-line.bin`。
3. **source 环境**：每次新终端都需要 `source install/setup.bash`（在项目根目录执行）。
4. **控制优先级**：急停 > 避障 > 导航 > 视觉 > 手动。任何自动控制都不能覆盖急停。
5. **实车先在仿真验证**：尽量先在本地仿真跑通逻辑，再上实车联调。
6. **保留人工接管**：自动控制运行中，确保能通过键盘或急停按钮立即接管。
