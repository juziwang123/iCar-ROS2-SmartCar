# 小车对接方案

> 基于《智能小车使用手册》第 3 章的官方流程，整理我们的控制链如何接入小车已有体系。

## 1. 小车软件体系总览

小车通过 Docker 运行两套容器：

### 1.1 容器 A：icar/ros-foxy:1.0.2（基础控制容器）

启动脚本：`~/run_docker.sh`  
快捷指令：`is`（启动）、`it`（停止）、`id`（进入）

| 指令 | 节点 | 包 | 用途 |
|------|------|----|------|
| `n1` | `Mcnamu_driver_X3` | `icar_bringup` | 底盘驱动 |
| `n2` | `sllidar_launch.py` | `sllidar_ros2` | 雷达驱动 |
| `n3` | `laser_Avoidance_a1_X3` | `icar_laser` | 雷达避障 |
| `n4` | `laser_Tracker_a1_X3` | `icar_laser` | 雷达跟随 |
| `n5` | `laser_Warning_a1_X3` | `icar_laser` | 雷达警卫 |
| `m1` | `astra.launch.xml` | `astra_camera` | 深度相机 |
| `m2` | `colorHSV` | `icar_astra` | HSV 颜色识别 |
| `m3` | `colorTracker` | `icar_astra` | 颜色追踪 |

### 1.2 容器 B：自动导航容器

启动脚本：`~/run_docker_autodrive.sh`  
快捷指令：`s`（启动）、`t`（停止）、`d`（进入）

**建图阶段：**

| 指令 | 命令 | 用途 |
|------|------|------|
| `m1` | `yahboomcar_nav map_gmapping_launch.py` | gmapping 建图 |
| `m2` | `yahboomcar_nav display_map_launch.py` | RViz 展示 |
| `m3` | `yahboomcar_ctrl yahboom_keyboard` | 键盘控制（键位：u/i/o/j/k/l/m/','/'.'） |
| `m4` | `yahboomcar_nav save_map_launch.py` | 保存地图 |

**导航阶段：**

| 指令 | 命令 | 用途 |
|------|------|------|
| `n1` | `yahboomcar_nav laser_bringup_launch.py` | 导航基础节点 |
| `n2` | `yahboomcar_nav display_nav_launch.py` | RViz 展示 |
| `n3` | `yahboomcar_nav navigation_dwa_launch.py` | DWA 导航 |
| `n4` | `yahboomcar_nav navigation_teb_launch.py` | TEB 导航 |

## 2. 待确认话题映射

部署前，**必须连上小车实际执行**以下命令确认话题名：

```bash
# 进入 iCar 容器
id

# 启动底盘驱动
n1

# 在另一个终端查看底盘订阅的话题
ros2 topic info /cmd_vel
```

同理确认：

| 节点 | 待确认项 | 预期结果 |
|------|---------|---------|
| `Mcnamu_driver_X3` | 订阅哪个话题接收速度 | 大概率 `/cmd_vel` |
| `laser_Avoidance_a1_X3` | 发布到哪个话题 | 大概率直接往 `/cmd_vel` 发 |
| `laser_Tracker_a1_X3` | 发布到哪个话题 | 同上 |
| `colorTracker` | 发布到哪个话题 | 同上 |
| `yahboom_keyboard` | 发布到哪个话题 | 同上 |
| DWA/TEB 导航 | 控制器发布到哪个话题 | 同上 |

## 3. 对接策略

### 3.1 核心思路

小车原有流程：多个节点直接往 `/cmd_vel` 发数据。  
对接后变成：

```
避障节点 ──→ /cmd_vel_lidar ──┐
跟随节点 ──→ /cmd_vel_follow ─┤
导航输出 ──→ /cmd_vel_nav ────┤
键盘控制 ──→ /cmd_vel_manual ─┼──→ safety_mux ──→ /control/cmd_vel ──→ motion_controller ──→ /cmd_vel ──→ 底盘驱动
视觉追踪 ──→ /cmd_vel_vision ──┤
雷达警卫 ──→ /cmd_vel_lidar ───┘
            （共用避障通道）
```

### 3.2 接入步骤

**步骤 1：进入 iCar 容器，确认话题**

```bash
id
n1    # 启动底盘
# 另开终端
id
ros2 topic list | grep cmd_vel
ros2 node info /Mcnamu_driver_X3
```

**步骤 2：启动 n3-n5 避障/跟随/警卫，确认输出话题**

```bash
# 终端 3
id && n3    # 避障
# 终端 4
id && ros2 topic echo /cmd_vel    # 观察是否有非零数据
```

**步骤 3：部署我们的控制包到 iCar 容器**

```bash
# 在开发机上
scp -r src/car_control jetson@<IP>:~/iCar-ROS2-SmartCar/
scp -r src/car_bringup jetson@<IP>:~/iCar-ROS2-SmartCar/

# 小车上，进入 iCar 容器
id
cd /path/to/car_control
colcon build --packages-select car_control car_bringup
source install/setup.bash
```

**步骤 4：启动我们的控制链，让原节点 remap 到我们的输入话题**

```bash
# 启动我们的控制链
ros2 launch car_bringup bringup.launch.py use_keyboard:=false

# 启动原车避障时，remap 输出到我们的输入话题
ros2 run icar_laser laser_Avoidance_a1_X3 --ros-args -r /cmd_vel:=/cmd_vel_lidar
```

如果小车原有 launch 文件不支持 remap，就需要修改对应的 launch 文件，或者用 `ros2 run` 手动指定。

**步骤 5：模式切换验证**

```bash
# 切到避障模式（此时 safety_mux 只接受雷达输入）
ros2 topic pub --once /mode_select std_msgs/msg/String "{data: manual}"  # 手动

# 切到导航模式
ros2 topic pub --once /mode_select std_msgs/msg/String "{data: nav}"

# 测试急停
ros2 topic pub --once /emergency_stop std_msgs/msg/Bool "{data: true}"
# 小车应立刻停止
ros2 topic pub --once /emergency_stop std_msgs/msg/Bool "{data: false}"
```

## 4. 最低风险对接方案

如果担心改动太大影响小车已有功能，先用最小侵入方式验证：

```bash
# 1. 只启动我们的 safety_mux，不启用任何控制源
ros2 run car_control safety_mux --ros-args -p default_mode:=manual

# 2. 确认 safety_mux 的输入话题和小车原节点的话题名一致
# 3. 手动发 cmd_vel_manual 确认能通过 safety_mux → motion_controller → /cmd_vel 控制小车
ros2 topic pub --once /cmd_vel_manual geometry_msgs/msg/Twist "{linear: {x: 0.1}, angular: {z: 0.0}}"

# 4. 确认无误后，再逐步 remap 原车节点到 safety_mux 的输入话题
```

## 5. 注意事项

1. **两个容器的话题隔离**：容器 A（iCar）和容器 B（导航）是独立容器，如果不在同一个 Docker 网络上，它们的话题互不可见。需要确认两个容器是否共享 ROS_DOMAIN_ID 和网络。

2. **Foxy 兼容性**：我们的代码已经改为 Python 3.8 兼容写法，可以在 Foxy 下运行。

3. **键盘键位不同**：小车键盘用 `u/i/o/j/k/l/m/,/.`，我们键盘用 `w/a/s/d`。建议保持小车原有键盘控制，我们的 `keyboard_teleop` 仅作备选。

4. **底盘驱动必须先启动**：`Mcnamu_driver_X3` 必须先于任何控制节点启动，否则 `/cmd_vel` 话题没有订阅者。

5. **固件模式影响控制接口**：手册 3.12 提到循线需要烧录特殊固件。不同的底盘固件可能影响控制接口的行为，对接前确认当前固件版本。
