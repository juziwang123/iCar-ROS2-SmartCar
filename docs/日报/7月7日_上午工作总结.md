## 4. 上午遇到的主要问题与处理结果

### 4.1 键盘控制在 launch 中无法使用

问题：`keyboard_teleop` 需要真实终端 TTY，直接挂在 `ros2 launch` 里会报 `termios.error: (25, 'Inappropriate ioctl for device')`。

处理结果：

1. 确认键盘控制不能作为普通 launch 子进程使用。
2. 最终改为“控制台直接发话题”作为当前更稳定的调试方式。

### 4.2 重复启动导致实体重名

问题：仿真未清理干净时重复启动，会出现 `Entity already exists`。

处理结果：

1. 明确后续调试应先清理旧仿真进程。
2. 当前推荐使用分终端独立启动方式，避免脚本式混合启动带来的干扰。

### 4.3 仿真导航无法直接使用

问题：上游 `wpr_simulation2` 缺少导航所需地图文件，导致 Nav2 场景无法直接完整启动。

处理结果：

1. 将优先级调整为先完成建图。
2. 地图保存后，再回到导航模式继续验证。

### 4.4 WSLg 图形性能问题

问题：RViz 和 Gazebo 同时打开时，比例异常、界面过大且非常卡。

处理结果：

1. 将建图默认流程改为更轻量的启动方式。
2. 优先使用简化场景与尽量少的 GUI 窗口进行本地联调。

## 5. 当前推荐启动与控制方式

目前最稳定的本地调试方式如下。

### 5.1 终端 1：启动仿真

```bash
cd /root/iCar-ROS2-SmartCar
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch wpr_simulation2 wpb_simple.launch.py
```

### 5.2 终端 2：启动控制链

```bash
cd /root/iCar-ROS2-SmartCar
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch car_bringup bringup.launch.py use_keyboard:=false use_mapping:=false use_navigation:=false use_patrol:=false use_lidar_avoidance:=false use_lidar_tracker:=false
```

### 5.3 终端 3：通过控制台发控制命令

```bash
cd /root/iCar-ROS2-SmartCar
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 topic pub --once /mode_select std_msgs/msg/String "{data: manual}"
ros2 topic pub --once /emergency_stop std_msgs/msg/Bool "{data: false}"
ros2 topic pub -r 10 /cmd_vel_manual geometry_msgs/msg/Twist "{linear: {x: 0.2}, angular: {z: 0.0}}"
```

停止命令：

```bash
ros2 topic pub --once /cmd_vel_manual geometry_msgs/msg/Twist "{linear: {x: 0.0}, angular: {z: 0.0}}"
```
